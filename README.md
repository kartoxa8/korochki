# Корочки.есть

Портал для записи на онлайн-курсы дополнительного профессионального
образования. Реализован фронтенд по образцу задания ГИА ДЭ
(модули 1–3). Бэкенд оставлен в виде заглушки на основе `localStorage` —
все методы API имитируют сетевые задержки и возвращают `Promise`, поэтому
их легко заменить на реальные HTTP-вызовы.

## Запуск

Не требует сборки. Откройте `index.html` напрямую в браузере или поднимите
любой статический сервер из корня проекта:

```bash
python3 -m http.server 8080
# затем откройте http://localhost:8080/
```

## Учетные записи для проверки

| Роль          | Логин      | Пароль     |
| ------------- | ---------- | ---------- |
| Администратор | `Admin`    | `KorokNET` |
| Пользователь  | `student1` | `demo1234` |

Демо-пользователь уже имеет 3 заявки в разных статусах для проверки
функциональности (включая возможность оставить отзыв о завершенном курсе).

## Структура

```
index.html              — главная страница с hero, слайдером и преимуществами
login.html              — авторизация
register.html           — регистрация
applications.html       — мои заявки (с отзывом для завершенных)
new-application.html    — формирование новой заявки
admin.html              — панель администратора с фильтрами и пагинацией

assets/
  css/style.css         — единый стиль приложения, адаптив 390×844 px
  js/api.js             — заглушка бэкенда (localStorage)
  js/app.js             — общая UI-инфраструктура (header, тосты, модалки)
  js/validators.js      — валидаторы полей форм
  js/slider.js          — слайдер с автопрокруткой 3 секунды
```

## Функционал

### Регистрация (`register.html`)
- Логин: латиница и цифры, минимум 6 символов
- Пароль: минимум 8 символов
- ФИО: кириллица и пробелы
- Телефон: маска `8(XXX)XXX-XX-XX`
- Email: стандартный формат
- Все ошибки валидации отображаются на форме под полем

### Авторизация (`login.html`)
- Проверка логина и пароля
- Ошибки выводятся под формой и в виде всплывающих сообщений
- Ссылка «Еще не зарегистрированы? Регистрация»

### Мои заявки (`applications.html`)
- Список заявок текущего пользователя
- Кнопка «Оставить отзыв» доступна только для заявок со статусом
  «Обучение завершено»
- Отзыв редактируется в модальном окне

### Новая заявка (`new-application.html`)
- Курс — выпадающий список из трех значений
- Дата начала — текстовое поле в формате `ДД.ММ.ГГГГ` с маской
- Способ оплаты — выбор: наличными или перевод по телефону
- После отправки заявка получает статус «Новая»

### Панель администратора (`admin.html`)
- Вход по `Admin` / `KorokNET`
- Таблица всех заявок со сведениями о пользователях
- Поиск по ФИО / логину / email / названию курса
- Фильтрация по статусу и курсу
- Пагинация (по 7 записей)
- Изменение статуса: Новая → Идет обучение → Обучение завершено
- Всплывающие сообщения о результатах действий

### Слайдер
4 слайда, автоматическая смена каждые 3 секунды, ручные кнопки
вперед/назад, точки-индикаторы, пауза при наведении.

## ER-диаграмма

```
┌────────────────────┐         ┌───────────────────────────┐
│       USERS        │ 1     N │        APPLICATIONS       │
├────────────────────┤◄────────┤───────────────────────────┤
│ id          PK     │         │ id              PK        │
│ login       UNIQUE │         │ user_id         FK USERS  │
│ password           │         │ course                    │
│ fio                │         │ start_date                │
│ phone              │         │ payment                   │
│ email              │         │ status                    │
│ created_at         │         │ review          NULLABLE  │
└────────────────────┘         │ created_at                │
                               │ updated_at                │
                               └───────────────────────────┘
```
